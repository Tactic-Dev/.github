## ðŸ› Reporting Bugs

Found a bug? Help us fix it!

1. **Check existing issues** - Someone may have already reported it
2. **Gather information**:
   - What were you trying to do?
   - What happened instead?
   - Steps to reproduce
   - Your environment (OS, version, etc.)
3. **Create a bug report** using the bug report template
4. **Be patient** - We'll review and respond as soon as possible

## âœ¨ Feature Requests

Have an idea for improvement?

1. **Check existing feature requests** - Your idea might already be there
2. **Describe your use case** - Help us understand the problem
3. **Suggest a solution** - Share your thoughts on implementation
4. **Submit a feature request** using the feature request template

## ðŸ”’ Security Issues

**Do NOT report security vulnerabilities publicly!**

For security issues:
1. Review our [Security Policy](SECURITY.md)
2. Use [GitHub Security Advisories](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing/privately-reporting-a-security-vulnerability) to report privately
3. Include detailed information about the vulnerability
4. Allow time for us to address the issue before public disclosure
